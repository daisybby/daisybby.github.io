抽象类和接口的区别

发表于 2024-06-25 更新于 2024-07-01 分类于 基础概念和原理 本文字数： 395 阅读时长 ≈ 1 分钟

方法定义：

  • 接口只是定义了一些方法；
  • 抽象类中方法是有具体实现代码的。

修饰符：

  • 接口中的方法只能被public修饰
  • 抽象类种的方法可以是private、protected和public，也可以是的。 构造器：
  • 接口不能有构造器；
  • 抽象类可以有构造器。

抽象类不能被实例化，但是构造器还是有意义的。抽象类的构造器可以在其子类的构造器中被调用，然后再执行子类自己的构造器代码。

继承与实现：

  • 接口只能被实现；
  • 抽象类可以被继承。

单继承，多实现：

  • 一个类可以实现多个接口
  • 一个类只能继承一个抽象类

职责不同：

  • 接口用于制定规范；
  • 抽象类用于代码复用。

在实际开发中，如果多个实现类有相同可复用的代码，那么就在接口和实现类之间增加一层抽象类。
