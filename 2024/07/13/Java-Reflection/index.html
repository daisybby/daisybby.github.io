Java反射机制

发表于 2024-07-13 更新于 2024-07-15 分类于 Java基础 本文字数： 466 阅读时长 ≈ 1 分钟

一、基本概念

反射机制是指程序在运行时能够获得自身的信息。
Java的反射机制可以实现：

  • 在运行时判断任意对象所属类；
  • 在运行时判断一个类所具有的成员变量和方法
  • 在运行时调用对象的方法
  • 在运行时构造任意类的对象

反射机制使得程序开发更加灵活，在运行时能够执行更多操作，但是反射机制也带来了一些问题:

  1. 代码可读性和可维护性降低
  2. 反射代码的执行性能低
  3. 反射破坏了封装性

二、为什么反射速度慢？

  1. 反射涉及到动态解析的类型，因此不能执行某些Java虚拟机优化
  2. 使用反射时，单数需要包装成Object[]类型，但是方法真正执行时又要解包；打包和解包的操作不仅耗费时间，中间还会产生很多对象，更容易导致GC；
  3. 反射调用方法时会从方法数组中遍历查找，检查方法可见性，这些操作都比较耗时；
  4. 参数额外检查。

三、反射常见的应用场景

  1. 动态代理
  2. BeanUtils中的属性拷贝
  3. RPC框架
  4. Spring的IOC/DI

四、反射和Class的关系

Java的Class类时反射机制的基础。被装入到JVM中的所有类，都有一个关联的Class对象，通过这个对象，可以实现被装入类的详细信息访问。
