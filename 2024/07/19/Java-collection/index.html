Java集合类和排序方式

发表于 2024-07-19 分类于 Java基础 本文字数： 1.9k 阅读时长 ≈ 2 分钟

一、Java集合类

常用的集合数据结构：Set、List、Queue、Stack和Map。
Set、List和Queue都继承自Collections接口。继承了Map接口的有：hashMap、hashTable和TreeMap。
Collection和Collections的联系与区别：

  • Collection：是集合类接口，提供了对集合类进行基本操作的通用接口。
  • Collections：是一个包含各种集合操作的静态方法的类，这个类并不能实例化，毕竟静态方法用类名.方法名就能调用。包装一个工具包。

Collection如何迭代？

  • 基于计数器的for循环：最简单的for(int i)循环。
  • 迭代器iterator：可以定义一个迭代器，基于顺序存储集合的迭代器可以按照位置直接访问，基于链式存储集合的迭代器需要保存当前遍历的位置，然后向前或者向后访问。
  • foreach循环：内部本质上也还是用迭代器iterator实现的。
  • 迭代器Enumeration：是iterator的古老版本。
  • Stream：JDK 1.8提供的Stream API也可以实现遍历。

List、Set、Queue和Map的区别？

  • List：存储的元素是有序的，可重复的。
  • Set：内部的元素是无序的，不可重复的。
  • Queue：保持先进先出的顺序，内部存储的元素是有序的，可重复的。
  • Map：存储的元素是KV键值对，key是唯一的，value是任意的。

三、扩展

compareTo和equals的使用场景有什么区别？

  • compareTo通常用于对象的排序，以及BigDecimal数值的比较；
  • equals通常用来比较两个对象是否相等。

Set是无序的，可以排序吗？
Set的无序并不是说必须维持无序的状态，而是说set中的插入顺序是无序的，但是还是可以通过以上方式对内部元素进行排序。
