一、Java注解的作用是什么？

Java注解并不直接影响代码的执行，不过有些注解可以用于这一目的。
通俗来讲，注解相当于一种标识，可以用来标识一个类、方法或者是字段。在其他地方的业务代码里，可以判断有某个注解的代码段，进而定制一些处理。
例如，@Bean注解，标识了要向IOC容器中注入一个Java Bean。

元注解就是用来定义其他注解的注解。

四个元注解：

  • @Target：表示注解可以用于修饰什么内容
  • @Retention：表示注解能够保存到什么级别
  • @Documented：将此注解包含到javadoc中
  • @Inherited：允许子类继承父类中的注解

2.1.@Retention

指定被修饰的注解的生命周期，有三个可选值：

  • SOURCE：注解仅在源代码保留，编译后就没了。
  • CLASS：注解在编译时保存到字节码中，但是在运行时不能被访问，仅用于编译时的检查。这是默认取值。
  • RUNTIME：注解在编译时保存到字节码中，并且在运行时可以通过反射进行访问，支持运行时检查注解信息。这也是最常用的取值。

2.2.@Target

指定被修饰的注解可以修饰什么元素类型，例如类、方法和字段等。这样可以限制注解的使用范围。
取值范围：

  • ElementType.CONSTRUCTOR: 表示注解可以应用于构造器。
  • ElementType.FIELD: 表示注解可以应用于字段（包括枚举的常量）。
  • ElementType.LOCAL_VARIABLE: 表示注解可以应用于局部变量。这个选项从Java 8开始支持。
  • ElementType.METHOD: 表示注解可以应用于方法。
  • ElementType.PACKAGE: 表示注解可以应用于包声明。
  • ElementType.PARAMETER: 表示注解可以应用于参数。
  • ElementType.TYPE: 表示注解可以应用于类、接口（包括注解类型）或枚举类型。
  • ElementType.ANNOTATION_TYPE: 表示注解可以应用于另一个注解类型。
  • ElementType.TYPE_PARAMETER: 表示注解可以应用于类型参数（泛型）。这个选项从Java 8开始支持。
  • ElementType.TYPE_USE: 表示注解可以应用于任何类型使用的地方，比如实例化对象、类型转换等。这个选项从Java 8开始支持。

2.3.@Documented

用于指示注解是否会出现在生成的Java文旦中，如果被@Documented元注解修饰，则该注解的信息会出现在API文档中。

2.4.@Inherited

指示该注解修饰的注解能不能被继承。默认情况下，注解不能被继承，也就是子类并不能继承父类的注解，但是@Inherited修饰的注解可以被子类继承。
