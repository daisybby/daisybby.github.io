run/start、wait/sleep、notify/notifyAll

发表于 2024-07-29 分类于 Java并发 本文字数： 672 阅读时长 ≈ 1 分钟

一、run()和start()方法区别

首先，调用start方法会开启一个新的线程进入就绪态，一旦获取了CPU资源，就自动执行run方法，run方法执行完，线程终止。
如果在一个线程中直接调用了run方法，那么就相当于是执行一个普通的方法，并不会开启新线程。
默认的方法是把需要并行处理的部分写到run方法里面，通过start方法开启新线程，然后自动调用run方法来并行执行。

在上一节讲到，创建线程的两种方法，一种是通过继承Thread类来实现，一种是实现Runnable接口来实现。通常通过实现Runnable接口来实现，因为Java是单继承，继承了Thread类就不能再继承其他的类了，而且Runnable只有一个run方法。

二、sleep和wait区别

sleep方法可以在任何地方用，但是wait只能在同步方法或者同步块中使用（和锁有关）。
wait会释放锁，sleep并不会。
wait方法执行后，线程会进入WAITING状态，直至被唤醒；sleep方法执行之后，线程进入TIMED_WAITING状态，等到指定时间之后再次尝试获取CPU时间片。

sleep是Thread类的方法，wait是Object类的方法。

三、notify和notifyAll区别

notify和notifyAll都可以唤醒等待队列中的线程。
使用notifyAll可以唤醒所有处于WAITING状态的线程，这些被唤醒的线程加入到锁的竞争队列中，可以去竞争对象锁，但是最终还是只有一个线程可以获取到对象锁。
notify和notifyAll都和锁有关，所以是Object类的方法。
