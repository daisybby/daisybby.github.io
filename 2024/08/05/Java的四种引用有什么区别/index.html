Java的四种引用有什么区别

发表于 2024-08-05 分类于 JVM 本文字数： 852 阅读时长 ≈ 1 分钟

JDK1.2之后，Java对引用的概念进行了扩充，将引用分为强引用、软引用、弱引用和虚引用四种（引用强度逐渐减弱）。

强引用：默认的引用方式，不需要我们显式定义。如果一个对象具有强引用，那么垃圾回收器一定不会回收这个对象。当内存空间不足的时候，JVM宁愿抛出OutOfMemoryError，终止应用程序，也不愿意回收掉具有强引用的对象。

软引用：软引用不能保证对象一定不会被回收，但是能够最大可能地保证。在内存空间充足的时候，只具有软引用的对象不会被回收；但是内存空间不足的时候，只具有软引用的对象是会被回收掉的。

弱引用：如果一个对象只具有弱引用，无论内存充足与否，只有弱引用的对象都会被回收掉。

虚引用：几乎没啥特殊的，如果一个对象仅有虚引用，那么就跟没有任何引用一样，在任何时候都可能被GC。虚引用主要是用在跟踪对象被垃圾回收的活动，比如写个日志啦之类。

弱引用的例子：之前提到的，ThreadLocal造成内存泄漏问题的解决方案，Entry中key对ThreadLocal对象的引用就是用弱引用，使得当ThreadLocal只有Entry这一个弱引用，而没有栈内存中的强引用之后，被GC就行了。
