MySQL中一条SQL语句执行的过程

发表于 2024-08-20 分类于 MySQL 本文字数： 484 阅读时长 ≈ 1 分钟

复杂的执行过程如下图所示：

看起来比较复杂，其实概括起来关键词就五个：连接器、查询缓存、解析器、查询优化器、执行器。

  1. SQL语句写好了，使用连接器，通过客户端和服务器的通信协议，连接上MySQL。还需要检查有没有权限使用。
  2. 在MySQL8.0之前，需要检查query cache，看看查询缓存里面有没有一模一样的SQL语句记录，如果命中，那直接返回查询结果。
  3. SQL语句被送到解析器进行语法解析，构建解析树。比如查询是select，表名是users，条件是age=’18’ and name=’Daisybby’。预处理器会根据MySQL规则进一步检查SQL语句是否合法，生成新的解析树。
  4. 优化器会生成执行计划，而且这个优化说的是会根据索引看看能不能进行优化。
  5. 执行器会执行SQL语句，会调用存储引擎来执行，得到查询的结果。如果开启了query cache，会将查询结果和SQL语句进行缓存，否则直接返回就行。
