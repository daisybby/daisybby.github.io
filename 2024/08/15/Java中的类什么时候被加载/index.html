Java中的类什么时候被加载

发表于 2024-08-15 分类于 JVM 本文字数： 375 阅读时长 ≈ 1 分钟

什么时候触发类的加载

在以下几种情况下，会触发类的加载：

  1. 当创建类的实例对象时，发现类还没有被加载，这个时候就会触发类的加载；
  2. 当调用类的静态方法、静态变量、或者静态代码块的时候，发现类还没加载，就会触发加载；
  3. 通过反射机制访问类的时候，类没有被加载，也会触发类的加载机制；
  4. 程序启动的时候，会主动加载一些基础类，或者是主体类。

什么时候触发类的卸载

满足以下两个条件时，类会被卸载：

  1. 类的所有实例对象都已经被回收
  2. 用于加载类的ClassLoader已经被回收

需要注意的是，尽管类已经可以被卸载了，JVM可能也不会立马就执行卸载，而是在JVM释放内存空间的时候才对不使用的类进行卸载。因为类卸载是一个比较耗时的操作，会影响程序的性能。

另外，在Java SE 9中，引入了模块的概念，通过模块化可以对类进行更加精细的控制，当模块中的类不再被引用的时候，这个模块就可以被卸载。
